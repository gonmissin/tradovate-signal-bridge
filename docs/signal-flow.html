Signal → execution → watch

Interactive map of the Tradovate client: webhook, TV market, v1 brackets, and the 5-second maintenance loop. Click any step to jump to details.

Flowchart

01 Webhook payload

Flask accepts JSON on /webhook. Only action: "ENTRY" is executed. Direction maps to buy/sell; instrument selects TV + optional v1 symbol via config.

  • Files: app.py → signal_to_execution_params
  • Skip path when regimeSuppressed and order.skip_when_regime_suppressed is true

02 Params & sizes

Stop and TP prices are resolved from the body and nested appLevels. Contract sizes come from assets/config.json (order_qty, tp1_qty, tp2_qty).

  • Two TPs require order.place_two_tps and tp1_qty + tp2_qty = order_qty
  • entry_price for BE = ask (buy) or bid (sell) at signal time

03 Authentication

JWT is obtained from Tradovate demo /auth/accesstokenrequest, cached ~80 minutes. Same token is used for v1 REST and the TV demo bridge headers.

  • modules/auth.py
  • Account row from /account/list (cached) for v1 accountId / accountSpec

04 Market execution

TradingView-style form POST to tv-demo.tradovateapi.com/accounts/{tv_account}/orders with type=market and static bid/ask from TV_BRIDGE_BID_ASK in client_config.py.

  • Orders._tv_market_order in modules/orders.py

05 Stops & targets

After a short fill wait, v1 /order/placeorder submits stop and limit leg(s). Responses should expose orderId for the watcher.

  • Single TP: full order_qty on one limit
  • Dual TP: split limits + shared initial stop on full size

06 Watcher registration

Bracket response JSON is parsed for SL/TP order ids. If parsing fails, log and skip — no background management for that trade.

  • modules/trade_watch.py · register_brackets

07 5-second poll

Daemon thread wakes every 5 seconds, calls GET /v1/order/list, matches ordStatus (Filled / Completed), then applies cancel/replace rules via POST /order/cancelorder and optional new stop.

08a TP1 hit

Cancel working stop. If tp2_qty > 0, place breakeven stop for runner size at entry_price. Single-TP: cancel stop and retire watch.

08b Stop hit

Before BE move: cancel TP1 and TP2. After BE move: cancel TP2 only (flat on scratch / runner stopped at entry).

08c TP2 hit

Target reached on runner — cancel any working stop so nothing is left dangling.

09 Configuration files

assets/config.json is the only config file: connection (login, tv_account) and order (sizes, instruments, place_two_tps, skip_when_regime_suppressed). TV bridge bid/ask placeholders live in modules/client_config.py (TV_BRIDGE_BID_ASK).

10 Lifecycle

Any first HTTP request triggers _ensure_trade_watch() so the daemon starts before or after the first trade. Use GET /health for uptime checks.
